# EverythingAgent 配置教程

本教程将指导您完成 EverythingAgent 的完整配置过程，包括环境准备、应用安装和 AI 模型配置。

## 🛠️ 环境准备

### 系统要求
- **操作系统**：Windows 10/11（推荐 64 位）
- **Node.js**：版本 18 或更高
- **内存**：至少 4GB RAM
- **存储空间**：至少 500MB 可用空间

### 必需软件安装

#### 1. 安装 Node.js
1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载 LTS 版本（推荐 v18.x 或更高）
3. 运行安装程序，使用默认设置
4. 验证安装：
   ```cmd
   node --version
   npm --version
   ```

#### 2. 安装 Everything 搜索工具
1. 访问 [Everything 官网](https://www.voidtools.com/)
2. 下载最新版本的 Everything
3. 安装并运行 Everything
4. 确保 Everything 在后台持续运行

### 开发环境优化（可选但推荐）

设置 npm 镜像以加速依赖安装：
```cmd
npm config set registry https://registry.npmmirror.com
```

设置 Electron 镜像（中国大陆用户）：
```cmd
set ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
set ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/
```

## 📦 应用安装

### 方法一：从源码构建（推荐开发者）

#### 1. 获取源码
```cmd
git clone <repository-url>
cd EverythingAgent
```

#### 2. 安装依赖
```cmd
npm install
```

#### 3. 图标配置（重要）
项目使用 `build/logo.png` 作为应用图标，构建时会自动转换为 Windows ICO 格式：
```cmd
npm run build:icon
```
此命令会：
- 读取 `build/logo.png`
- 生成多尺寸 ICO 文件
- 输出到 `build/icon.ico`

#### 4. 启动开发模式
```cmd
npm run dev
```

应用启动后会自动打开主窗口。

#### 5. 生产构建
```cmd
# 仅构建前端和 Electron
npm run build:vite

# 完整构建（自动处理图标 + 生成安装包）
npm run build
```

构建完成后，安装包位于 `release/` 目录下。

#### 6. 版本升级说明
安装新版本时：
- ✅ **自动卸载旧版本**：安装程序会自动检测并卸载旧版本
- ✅ **保留用户配置**：个人设置和对话历史会被保留
- ✅ **无缝升级**：无需手动卸载，直接运行新安装包即可

升级流程：
1. 运行新版本安装包
2. 安装程序检测到旧版本
3. 自动卸载旧版本
4. 安装新版本
5. 保留原有配置文件

### 方法二：直接使用预编译版本

如果已有 `.exe` 安装包：
1. 双击安装包运行安装向导
2. 按提示完成安装
3. 从开始菜单或桌面快捷方式启动应用

## 🎨 应用图标配置

### 图标文件要求
- **文件格式**：PNG 格式
- **推荐尺寸**：512x512 像素（正方形）
- **文件位置**：`build/logo.png`
- **透明背景**：建议使用透明背景以获得最佳效果

### 自动图标转换
项目包含自动图标转换功能：

1. **开发阶段**：
   ```cmd
   npm run build:icon
   ```
   - 将 `build/logo.png` 转换为 Windows ICO 格式
   - 生成多尺寸图标（16x16 到 256x256）
   - 输出到 `build/icon.ico`

2. **构建阶段**：
   ```cmd
   npm run build
   ```
   - 自动执行图标转换
   - 集成到最终安装包中

### 手动图标替换
如果需要更换图标：

1. 准备新的 PNG 图标文件
2. 替换 `build/logo.png`
3. 重新运行图标转换：
   ```cmd
   npm run build:icon
   ```
4. 重新构建应用以应用新图标

### 图标最佳实践
✅ **推荐做法**：
- 使用高分辨率 PNG（512x512 或更高）
- 保持简洁的设计风格
- 确保在小尺寸下仍清晰可辨
- 使用透明背景

❌ **避免做法**：
- 使用复杂细节过多的设计
- 选择对比度低的颜色组合
- 使用 JPEG 等有损压缩格式
- 忽略小尺寸显示效果

### 图标应用范围
转换后的图标将应用于：
- 🖼️ **应用程序窗口**：任务栏和 ALT+TAB 切换时显示
- 🖥️ **桌面快捷方式**：安装后创建的桌面图标
- 📦 **安装程序**：安装包本身的图标
- 🗂️ **开始菜单**：程序列表中的图标
- 🗑️ **卸载程序**：控制面板中的卸载图标

## ⚙️ 基础配置

### 首次启动设置

应用首次启动时会自动进行基础配置：

1. **窗口位置**：默认居中显示
2. **快捷键**：默认 `Alt+Space`
3. **系统托盘**：自动启用
4. **失焦隐藏**：默认开启

### 通用设置配置

通过以下方式打开设置面板：
- 点击窗口右上角齿轮图标 ⚙️
- 右键系统托盘图标选择「设置」

#### 可配置选项：

**快捷键设置**
- 修改全局唤醒快捷键
- 支持组合键（如 Ctrl+Shift+Space）
- 注意避免与其他应用冲突

**窗口行为**
- 失焦时是否自动隐藏
- 启动时是否自动显示
- 窗口透明度调节

**搜索设置**
- 默认搜索路径配置
- 搜索结果显示数量
- 文件类型过滤选项

## 🤖 AI 模型配置

这是最重要的配置环节，决定了 AI 功能的可用性。

### 添加第一个 AI 模型

#### 步骤 1：进入模型配置页面
1. 打开设置面板
2. 点击「模型配置」选项卡
3. 点击「添加模型」按钮

#### 步骤 2：选择供应商类型

目前支持以下供应商：

**OpenAI 兼容（推荐新手）**
- 适用于：DeepSeek、OneAPI、NewAPI 等中转平台
- 配置简单，兼容性好

**通义千问 DashScope**
- 适用于：阿里云通义千问系列模型
- 需要阿里云账号和 API Key

**阿里云百炼**
- 适用于：阿里云百炼平台模型
- 企业级 AI 服务

**Anthropic**
- 适用于：Claude 系列模型
- 需要 Anthropic 账号

**Google AI**
- 适用于：Gemini 系列模型
- 需要 Google AI Studio 账号

#### 步骤 3：填写配置信息

以 **DeepSeek + OneAPI** 为例：

**基本信息**
- 模型名称：`DeepSeek Chat`
- 供应商类型：选择「OpenAI 兼容」

**连接配置**
- Base URL：`https://api.deepseek.com/v1` （或你的 OneAPI 地址）
- API Key：你的 DeepSeek API 密钥
- 模型名称：`deepseek-chat`

**高级设置**
- 最大 Token 数：4096（根据需要调整）
- 温度参数：0.7（控制回复随机性）
- 超时时间：30 秒

#### 步骤 4：测试连接
1. 点击「测试连接」按钮
2. 等待测试结果
3. 成功后点击「保存」

#### 步骤 5：激活模型
1. 在模型列表中找到刚添加的模型
2. 点击模型左侧的空心圆圈
3. 圆圈变为绿色实心表示已激活

### 多模型管理

#### 添加备用模型
建议配置 2-3 个不同供应商的模型作为备用：
1. 主力模型：日常使用的主要模型
2. 备用模型：当主力模型不可用时切换
3. 特殊用途模型：针对特定任务的专用模型

#### 模型切换方法
1. 打开设置 → 模型配置
2. 点击想要激活的模型左侧圆圈
3. 当前激活的模型会显示绿色实心圆圈

### 常见配置问题解决

#### API Key 相关问题
**问题**：API Key 无效
**解决方案**：
- 检查 Key 是否正确复制
- 确认账户余额充足
- 验证服务商地区限制

**问题**：Key 泄露风险
**解决方案**：
- 本应用所有密钥本地加密存储
- 建议定期更换 API Key
- 不要在公共场合展示配置界面

#### 连接问题
**问题**：无法连接到 API
**解决方案**：
- 检查网络连接
- 验证 Base URL 是否正确
- 确认防火墙未阻止应用
- 尝试使用代理服务器

#### 模型相关问题
**问题**：模型不支持某些功能
**解决方案**：
- 查阅对应模型官方文档
- 调整最大 Token 参数
- 更换其他兼容模型

## 🔧 高级配置

### 自定义快捷键
支持复杂的快捷键组合：
- `Ctrl+Alt+Space`
- `Win+Shift+A`
- `F12`

注意事项：
- 避免与系统快捷键冲突
- 某些组合可能被其他软件占用
- 建议重启应用使快捷键生效

### 代理设置
如果需要通过代理访问 AI 服务：

1. 在模型配置中设置代理 URL
2. 格式：`http://proxy-server:port`
3. 或使用系统代理设置

### 数据目录管理
应用数据存储位置：
- Windows：%APPDATA%\EverythingAgent\
- 包含配置文件和对话历史

可以定期备份此目录以防数据丢失。

## 🛡️ 安全配置建议

### API Key 安全
1. **定期更换**：建议每月更换一次 API Key
2. **权限控制**：为不同模型设置不同的 Key
3. **使用限制**：在服务商处设置使用限额

### 系统安全
1. **防火墙**：允许应用通过防火墙
2. **杀毒软件**：将应用目录加入白名单
3. **权限管理**：以普通用户权限运行

## 📊 性能优化配置

### 内存优化
- 关闭不必要的后台模型
- 适当减少同时加载的模型数量
- 定期清理对话历史

### 启动优化
- 设置开机自启动（可选）
- 调整启动时的行为配置
- 预加载常用模型

## 🔧 故障排除

### 常见问题及解决方案

**问题 1：应用无法启动**
```
解决方案：
1. 检查 Node.js 版本是否符合要求
2. 重新安装依赖：npm install
3. 查看错误日志获取详细信息
```

**问题 2：快捷键不生效**
```
解决方案：
1. 检查快捷键是否被其他程序占用
2. 重启应用
3. 在设置中重新设置快捷键
```

**问题 3：搜索功能异常**
```
解决方案：
1. 确认 Everything 已正常运行
2. 检查 es.exe 文件是否存在
3. 重启 Everything 服务
```

**问题 4：AI 回复缓慢**
```
解决方案：
1. 检查网络连接质量
2. 尝试切换到其他模型
3. 调整超时设置
4. 检查服务商状态
```

### 日志查看
开发模式下可通过以下方式查看日志：
1. 按 `Ctrl+Shift+I` 打开开发者工具
2. 切换到 Console 标签页
3. 查看实时日志输出
